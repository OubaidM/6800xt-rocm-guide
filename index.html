Training YOLOv8 on AMD RX6800XT with ROCm

A comprehensive guide to configuring, training, and optimizing computer vision models on AMD hardware

AMD ROCm 6.0.2 Ubuntu 22.04.05 LTS RX6800XT RDNA2

Introduction

This guide provides a comprehensive, step-by-step process for training the Ultralytics YOLOv8 object detection model on a custom dataset using an AMD Radeon RX6800XT GPU. The setup is specifically tailored for Ubuntu 22.04.05 LTS and leverages the AMD ROCm (Radeon Open Compute) software stack to enable high-performance GPU acceleration.

Key Requirements

  • • AMD Radeon RX6800XT GPU (RDNA2 architecture)
  • • Ubuntu 22.04.05 LTS operating system
  • • Labeled dataset in YOLOv8 format
  • • Basic familiarity with terminal commands

The instructions cover the entire workflow, from the initial environment setup and software installation to dataset preparation, model training, and performance optimization techniques designed to maximize the potential of your RX6800XT.

Environment Setup for AMD ROCm

The foundation for training YOLOv8 on an AMD GPU is a correctly configured ROCm environment. This section details the critical steps for installing the ROCm drivers and libraries, configuring the system specifically for the RX6800XT's RDNA2 architecture, and installing a compatible version of PyTorch.

Installing ROCm Drivers and Libraries

The installation of the ROCm stack is the first and most critical step. It involves downloading the official installer from AMD, executing the installation with specific flags to ensure stability, and verifying the installation.

1. Downloading the AMDGPU-Installer Package

The recommended method for installing ROCm on Ubuntu is to use the official amdgpu-install package provided by AMD. For Ubuntu 22.04, a stable version is ROCm 6.0.2, which corresponds to the Radeon Software for Linux version 23.40.2 [63]. 

# Download the AMDGPU installer package
wget https://repo.radeon.com/amdgpu-install/23.40.2/ubuntu/jammy/amdgpu-install_6.0.60002-1_all.deb

# Install the package
sudo apt install ./amdgpu-install_6.0.60002-1_all.deb

2. Installing ROCm with DKMS Override

To avoid potential conflicts with existing kernel drivers, it's recommended to install ROCm without modifying the existing kernel modules using the --no-dkms flag [63]. 

# Install ROCm without kernel module modifications
sudo amdgpu-install --usecase=rocm --no-dkms

3. Adding User to Required Groups

Add your user account to the render and video groups to grant GPU access permissions [63]. 

# Add user to required groups
sudo usermod -a -G render $LOGNAME
sudo usermod -a -G video $LOGNAME

# Reboot required for changes to take effect
sudo reboot

4. Verifying ROCm Installation

After rebooting, verify the installation using the rocm-smi command [63]. 

# Check GPU status and ROCm installation
rocm-smi

Configuring the Environment for RX6800XT (RDNA2)

The RX6800XT is based on AMD's RDNA2 architecture and requires specific configuration to ensure full compatibility and optimal performance.

Setting HSA_OVERRIDE_GFX_VERSION

Set the HSA_OVERRIDE_GFX_VERSION environment variable to 10.3.0 for RDNA2 compatibility [61] [66]. 

# Add to ~/.bashrc for persistence
echo 'export HSA_OVERRIDE_GFX_VERSION=10.3.0' >> ~/.bashrc
source ~/.bashrc

Isolating the Dedicated GPU

In systems with multiple GPUs, ensure ROCm applications use the RX6800XT by setting visibility variables [61]. 

# Add to ~/.bashrc to isolate RX6800XT
echo 'export HIP_VISIBLE_DEVICES=0' >> ~/.bashrc
echo 'export ROCR_VISIBLE_DEVICES=0' >> ~/.bashrc
source ~/.bashrc

Installing PyTorch with ROCm Support

Install PyTorch specifically compiled for ROCm to enable AMD GPU acceleration.

Installing PyTorch for ROCm 6.0

Use the custom PyTorch repository for ROCm 6.0 compatibility [66]. 

# Install PyTorch with ROCm support
pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/rocm6.0

Verifying PyTorch Installation

Verify that PyTorch can detect and use your RX6800XT GPU. 

# Test PyTorch GPU recognition
python3 -c "import torch; print(f'ROCm available: {torch.cuda.is_available()}'); print(f'Device name: {torch.cuda.get_device_name() if torch.cuda.is_available() else "None"}')"

Setting Up YOLOv8 and Your Custom Dataset

With the deep learning environment fully configured, the next phase involves setting up the YOLOv8 framework and preparing your custom dataset for training.

Installing the Ultralytics YOLOv8 Framework

Installing the Official Package

Install the official ultralytics package from PyPI [57]. 

# Install YOLOv8 framework
pip install ultralytics

Preparing Your Labeled Dataset

Dataset Directory Structure

Organize your dataset according to YOLOv8's expected structure [57]. 

dataset/
├── images/
│   ├── train/
│   │   ├── image_001.jpg
│   │   └── image_002.jpg
│   └── val/
│       ├── image_101.jpg
│       └── image_102.jpg
└── labels/
    ├── train/
    │   ├── image_001.txt
    │   └── image_002.txt
    └── val/
        ├── image_101.txt
        └── image_102.txt

Creating data.yaml Configuration File

Create a YAML configuration file to specify dataset paths and classes [57]. 

# data.yaml example
path: /path/to/your/dataset
train: images/train
val: images/val

nc: 2  # number of classes
names: ['cat', 'dog']

Running the Training Process

With the environment prepared and dataset configured, you are now ready to start training your custom YOLOv8 model.

Constructing the Training Command

Basic Training Command

Execute the training command with essential parameters [57]. 

# Start YOLOv8 training
yolo train data=path/to/your/data.yaml model=yolov8n.pt epochs=100 imgsz=640

Explicit Task Specification

For clarity, you can explicitly specify the detection task [55]. 

# Explicit detection training command
yolo detect train data=path/to/your/data.yaml model=yolov8n.pt epochs=100 imgsz=640

Monitoring Training Progress

Real-Time Metrics

Monitor training progress through terminal output showing loss values, precision, recall, and mAP metrics [59].

Training Results

Training results, model checkpoints, and visualizations are saved in the runs/detect/train directory [59].

  • best.pt - Best performing model weights
  • last.pt - Final epoch model weights
  • results.png - Training metrics visualization

Optimizing Performance for the RX6800XT

To fully leverage the power of the RX6800XT, it's important to apply specific optimization techniques designed for ROCm and your GPU architecture.

Leveraging ROCm-Specific Features

Composable Kernel (CK) Acceleration

CK provides highly optimized GPU kernels for deep learning operations. Enable ROCm backend for Triton compiler [61]. 

# Enable ROCm backend for Triton
export TRITON_USE_ROCM=ON

Mixed Precision Training

Utilize FP16 mixed precision training for significant speedup and memory reduction [61].

PyTorch's autocast context manager automatically handles mixed precision training. This feature is particularly effective on modern GPUs like the RX6800XT.

Advanced Training Techniques

Multi-GPU Training with DDP

Scale training across multiple AMD GPUs using Distributed Data Parallel for faster processing. 

# Multi-GPU training example
yolo train data=path/to/data.yaml model=yolov8n.pt epochs=100 imgsz=640 device=0,1,2,3

Optimal Batch Size Configuration

Adjust batch size to maximize VRAM usage on your RX6800XT (16GB). Monitor memory usage with rocm-smi during training. 

# Monitor GPU memory usage during training
watch -n 1 rocm-smi

Troubleshooting Common Issues

Resolving Segmentation Faults

Segmentation faults often indicate compatibility issues. Verify your environment setup [61].

Common Solutions:
  • • Verify HSA_OVERRIDE_GFX_VERSION=10.3.0 is set
  • • Ensure PyTorch is installed for the correct ROCm version
  • • Check GPU driver compatibility
  • • Validate dataset paths and formats

Addressing Performance Bottlenecks

Low GPU utilization may indicate system bottlenecks. Common causes include:

Data Loading

Increase workers parameter for faster data loading

Storage Speed

Move dataset to SSD for improved read performance

Model Size

Use larger models (yolov8s/l/x) to better utilize GPU

Batch Size

Increase batch size to maximize memory usage
